About Evder

Evder is a website developed individually by me, for a class project.
It was built using html, css, php, ajax and mysql.

What is Evder?

Evder was developed as an attempt to solve a problem. For the project, we were required to only fulfill a set of technical requirements, the topic and content of our websites were left up to us. I wanted to come up with an attempt to challenge Vancouver's image as a city without much excitement to it.

From this came the idea of Evder: a website that collects all of the events that are happening in a city of your choice and displays them to you in one location. Users are able to sort the type of event they would like to go to by the city, type of event and price of the event.

The Process

Before I began coding anything, I created a quick mockup of my preferred layout in adobe illustrator.

The Database

This was my first time working with backend development and so developing my database design was an iterative process. Originally, I believed my database would be quite small, after all, I only needed to manage events and users. But as I began coding, I began to notice the need to add more and more tables to my database. Upon completion, I had five: events, personalization, posts, savedposts, and users.

Challenges

One of my most memorable challenges was when I was attempting to implement the ability for users to save events. I was attemping to save multiple userid's for a single event, in the events table. This was because I wanted to keep track of which events were saved to the accounts of which users.

It was not until later, when saying aloud my problem, that I realized that I was going about it all wrong. This lead to the creation of a new table in my database which stores the username of the user, along with the event id, to be able to access that event later.

Reflection

If I were to do this again, I spend more time working on the design of my website as, at the time, I did not have enough time to work on the frontend side of the things. At the time, I had no experience with frameworks such as Bootstrap or React to have made designing the website easier.

As the site is now, I’m proud that I managed to get all of the functionalities required to work as it took me a while to learn how to do so. Overall, I think I ended up learning a lot by struggling through various problems and figuring out how to solve them. Some parts definitely required me to step back and reexamine the issue in order to solve the problem and I feel that this made me more capable of handling such problems in the future.
